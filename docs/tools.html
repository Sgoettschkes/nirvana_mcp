nirvana-mcp · Tool Reference

The MCP server @sgoettschkes/nirvana-mcp exposes the following 17 tools to Claude Code and any MCP-compatible client. All tools are read-only against NirvanaHQ.

Setup: npx -y @sgoettschkes/nirvana-mcp login prints an auth token. Then claude mcp add nirvana --env NIRVANA_AUTH_TOKEN=… -- npx -y @sgoettschkes/nirvana-mcp to register the server.

Tool Inputs Returns Description
State-filtered lists
list_inbox none [{id, name, note, tags[]}] Tasks in the Inbox (state=0) — unprocessed items the user hasn't categorized yet.
list_next_actions none [{id, name, note, tags[]}] Tasks marked Next (state=1) — concrete, actionable next steps the user has committed to.
list_waiting_for none [{id, name, note, tags[]}] Tasks Waiting on someone else (state=2) — blocked on a contact's response.
list_scheduled none [{id, name, note, tags[]}] Tasks deferred to a specific future start date (state=3).
list_someday none [{id, name, note, tags[]}] Tasks parked in Someday/Maybe (state=4) — revisit later, not actionable now.
list_later none [{id, name, note, tags[]}] Tasks in the Later list (state=5) — between Next and Someday.
list_trash none [{id, name, note, tags[]}] Soft-deleted tasks (state=6) — still recoverable from Trash.
list_logbook none [{id, name, note, tags[]}] Completed tasks (state=7). Currently returns full history — date-range filter is a planned enhancement.
list_recurring none [{id, name, note, tags[]}] Recurring task templates (state=9) — the rules that spawn instances on a schedule, not the individual occurrences.
Flag & type filters
list_focus none [{id, name, note, tags[]}] Tasks flagged for Focus (seqt > 0), excluding completed or trashed items — what the user is actively prioritizing.
list_projects none [{id, name, note, tags[]}] Active projects (type=1, state=11) — containers grouping related tasks under a shared outcome.
get_project id?: string name?: string Exactly one required. { project: {...}, children: [{id, name, note, tags[], type, state, completed}] } A single project plus its direct child tasks and sub-projects. Use list_projects to discover names.
Taxonomy
list_areas none [{name, color?}] High-level life domains (e.g. work, personal) that group projects and tasks.
list_tags none [{name, kind: "tag"|"context"|"contact", email?, color?}] Plain tags, GTD contexts (e.g. @phone) and contacts (people you're waiting on), distinguished by kind.
list_reference_lists none [{id, name, note, tags[]}] Reference Lists (type=3, state=10) — top-level containers for non-actionable notes and lookup material.
get_reference_list id?: string name?: string Exactly one required. { reference_list: {...}, items: [{id, name, note, tags[]}] } A single Reference List plus all of its child items (type=2, state=10). Mirrors get_project.
Search
search_tasks text?: string tag?: string area?: string state?: enum At least one required. [{id, name, note, tags[], state, completed}] Filter tasks across the whole account. Trashed/deleted are always excluded; completed (logged) tasks are excluded unless state: "logged" is passed explicitly.
